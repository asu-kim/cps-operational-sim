# Drone Data

## Overview

This directory stores the drone CSV data used by the simulation, plotting, and comparison workflow.

The `rc-out/` folder contains the RC command logs generated by the Lingua Franca drone code.

The raw ToF sensor logs are stored separately and are paired with the matching RC output file when generating plots or metrics.

## Directory Layout

```text
data/drone/
  README.md
  rc-out/
    rc-out.csv
    rc-out-2.csv
    rc-out-3.csv
    rc-out-4.csv
  raw-logs/
    Data1/
      front.csv
      left.csv
      right.csv
      top.csv
      bottom.csv
    Data2/
      front.csv
      left.csv
      right.csv
      top.csv
      bottom.csv
```

The exact run names can vary, but each ToF data directory should contain the five sensor CSV files:

```text
front.csv
left.csv
right.csv
top.csv
bottom.csv
```

## RC Output Logs

The files in `rc-out/` are generated by the Lingua Franca drone workflow. Typical files include:

- `rc-out.csv`: original or first RC command run
- `rc-out-2.csv`: modified or second RC command run
- `rc-out-3.csv`, `rc-out-4.csv`: additional runs

Each RC output file should be paired with the corresponding ToF sensor directory. For example:

```text
rc-out/rc-out.csv    -> raw-logs/Data1/
rc-out/rc-out-2.csv  -> raw-logs/Data2/
```

## Generate a Single-Run Plot

From the repository root:

```bash
python3 tools/plot_drone_path.py \
  data/drone/rc-out/rc-out.csv \
  data/drone/raw-logs/Data1
```

This creates a path and obstacle plot for one drone run.

## Generate a Two-Run Comparison Plot

From the repository root:

```bash
python3 tools/plot_drone_path_compare.py \
  data/drone/rc-out/rc-out.csv data/drone/raw-logs/Data1 \
  data/drone/rc-out/rc-out-2.csv data/drone/raw-logs/Data2 \
  results/drone/simulation/original_vs_modified_overlay.pdf
```

The comparison plot overlays the original and modified trajectories on the same axes. The plotting script reconstructs an approximate drone path from the RC commands and overlays obstacle detections from the ToF sensor logs.

## Quantify Original vs Modified Results

Use the drone quantification script to compare duration, path length, obstacle detections, ToF clearance, and control effort:

```bash
python3 tools/quantify_drone_results.py \
  --rc-original data/drone/rc-out/rc-out.csv \
  --data-original data/drone/raw-logs/Data1 \
  --rc-modified data/drone/rc-out/rc-out-2.csv \
  --data-modified data/drone/raw-logs/Data2 \
  --out results/drone/simulation/drone_quantification.csv
```

Useful metrics include:

- duration of each run
- reconstructed path length
- net displacement from start to end
- straightness percentage
- number of detected obstacle points
- minimum, mean, and percentile ToF clearance
- control effort from RC commands
- trajectory separation between original and modified runs

## Notes

- Keep RC command logs in `data/drone/rc-out/`.
- Keep raw ToF sensor logs in `data/drone/raw-logs/`.
- Save generated plots and metrics under `results/drone/simulation/`.
- The reconstructed path is approximate because it is integrated from RC commands rather than measured with an external motion-capture system.
- Use the plots and metrics for relative comparison between runs, not as absolute ground-truth localization.
