# CPS Operational Simulation

## Overview

This repository contains the implementation and experimental data for a deterministic, operational-data-driven cyber-physical system (CPS) simulation workflow. The workflow records structured operational data from resource-constrained robots and replays those logs to reconstruct and compare robot executions.

The repository follows the terminology used in the paper draft:

- **Track following** with the Pololu 3Pi+ 2040 robot on square and stadium tracks.
- **Bump-based maze exploration** with the Pololu 3Pi+ 2040 robot.
- **Drone obstacle avoidance** using five time-of-flight (ToF) sensor streams and RC command outputs.

The main components are:

- [`pololu-3pi/`](pololu-3pi/README.md): LF programs for collecting operational data from the Pololu robot.
- [`mujoco-pololu-replay/`](mujoco-pololu-replay/README.md): LF + MuJoCo replay programs for track-following and maze-exploration logs.
- [`drone/`](drone/README.md): LF drone obstacle-avoidance workflow using live or recorded ToF data.
- [`data/`](data/README.md): raw logs and RC-output logs used by the experiments.
- [`tools/`](tools/README.md): Python plotting tools for drone trajectory and obstacle visualizations.
- [`results/`](results/README.md): generated PDFs and experiment outputs.

## Setup Instructions

The repository has three setup layers:

1. General LF tooling used by all workflows.
2. RP2040/Pico tooling used by the Pololu 3Pi+ 2040 programs.
3. MuJoCo tooling used by the Pololu replay simulations.

The commands below are for Ubuntu or WSL Ubuntu. macOS setup is intentionally omitted.

### 1. General Ubuntu Packages

Install the common tools used by the LF embedded workflow:

```bash
sudo apt update
sudo apt install gh git curl openjdk-17-jdk openjdk-17-jre screen cmake make gcc
```

Install Visual Studio Code if you want the LF and C/C++ editor support:

```bash
sudo snap install code --classic
```

Useful VS Code extensions are:

```bash
code --install-extension ms-vscode.cmake-tools
code --install-extension ms-vscode.cpptools
code --install-extension lf-lang.vscode-lingua-franca --pre-release
code --install-extension marus25.cortex-debug
```

### 2. Install Lingua Franca CLI Tools

Install the nightly Lingua Franca command-line tools, including `lfc`, `lfd`, and `lff`:

```bash
curl -Ls https://install.lf-lang.org | bash -s nightly cli
```

Restart the shell or reload your profile if `lfc` is not immediately found. Check the installation with:

```bash
lfc --version
```

If the installer reports a permission error while creating files under `/usr/local/share/lingua-franca`, use the sudo installation path:

```bash
wget https://raw.githubusercontent.com/lf-lang/installation/main/install.sh
chmod +x install.sh
sudo bash install.sh nightly cli
```

### 3. Clone This Repository and Initialize Submodules

From the parent directory where you want the repository:

```bash
git clone git@github.com:asu-kim/cps-operational-sim.git
cd cps-operational-sim
git submodule update --init --recursive
```

The important submodules are:

```text
pololu-3pi/pico-sdk
mujoco-c
```

### 4. Pololu 3Pi+ 2040 / RP2040 Non-Nix Setup

The Pololu programs target the RP2040 on the Pololu 3Pi+ 2040 robot. If you are not using Nix, install the Pico SDK, `picotool`, the ARM cross compiler, and the bare-metal C library manually.

Install the ARM/RP2040 build dependencies:

```bash
sudo apt update
sudo apt install cmake make gcc gcc-arm-none-eabi libnewlib-arm-none-eabi
```

This repository already includes the Pico SDK as a submodule. Point `PICO_SDK_PATH` to it:

```bash
cd ~/cps-operational-sim
git submodule update --init --recursive pololu-3pi/pico-sdk
export PICO_SDK_PATH=$PWD/pololu-3pi/pico-sdk
echo "export PICO_SDK_PATH=$PWD/pololu-3pi/pico-sdk" >> ~/.profile
```

Build and install `picotool`:

```bash
cd ~/cps-operational-sim
git clone https://github.com/raspberrypi/picotool.git external/picotool
cmake -S external/picotool -B external/picotool/build
cmake --build external/picotool/build
sudo cmake --install external/picotool/build
```

Allow non-root USB access to RP2040 devices:

```bash
curl -s https://raw.githubusercontent.com/raspberrypi/picotool/master/udev/60-picotool.rules | sudo tee -a /etc/udev/rules.d/60-picotool.rules >/dev/null
sudo udevadm control --reload-rules
sudo udevadm trigger
```

After this, unplug and reconnect the Pololu robot if it was already connected.

## Pololu Operational-Data Collection

Build the Pololu LF programs from the repository root:

```bash
lfc pololu-3pi/src/BumpMaze_ENCODER_buttons.lf
lfc pololu-3pi/src/TrackFollowSolution_encoder_imu.lf
```

Use the PowerShell loggers in `pololu-3pi/` to capture USB serial CSV output:

```powershell
cd C:\path\to\cps-operational-sim\pololu-3pi

.\log-pololu-bump-encoder.ps1 -PortName COM5 -BaudRate 115200 -TestName pololu_bump_encoder_maze
.\log-pololu-encoder.ps1 -PortName COM5 -BaudRate 115200 -TestName pololu_line_encoder_imu_track_follow
```

Save captured CSV logs under:

```text
data/pololu-3pi/raw-logs/
```

## MuJoCo Replay for Pololu Logs

The MuJoCo replay project uses fixed CSV filenames in `mujoco-pololu-replay/data/`:

```text
maze_log.csv
racetrack_log.csv
square_log.csv
```

Build and run from the replay directory:

```bash
cd mujoco-pololu-replay
rm -rf build
lingo update
lingo build
export LD_LIBRARY_PATH=/opt/mujoco/lib:$LD_LIBRARY_PATH

build/bin/MazeTraceReplay
build/bin/RacetrackTraceReplayOval
build/bin/RacetrackTraceReplaySquare
```

Use the generator scripts in `mujoco-pololu-replay/tools/` to refresh measured ground-truth XML files when physical dimensions change.

## Drone Obstacle-Avoidance Workflow

The drone workflow uses five ToF sensor CSV files:

```text
front.csv
left.csv
right.csv
top.csv
bottom.csv
```

RC command logs generated by the LF drone simulation are stored in:

```text
data/drone/rc-out/
```

Generate a two-run comparison plot:

```bash
python3 tools/plot_drone_path_compare.py data/drone/rc-out/rc-out.csv data/drone/raw-logs/Data1 data/drone/rc-out/rc-out-2.csv data/drone/raw-logs/Data2 results/drone/simulation/original_vs_obstacle_injected_overlay.pdf
```

## Data and Results Policy

- Keep raw logs unchanged.
- Store generated RC command logs under `data/drone/rc-out/`.
- Store Pololu robot logs under `data/pololu-3pi/raw-logs/`.
- Store generated figures and reports under `results/`.
- If a script repairs a CSV header or filters data, write a new file instead of overwriting the raw log.

## Notes

Some LF files may contain machine-specific paths or data filenames. If you move the repository, check and update the configured CSV paths before running the LF programs.
