# Drone CSV Simulation

## Overview

This directory contains the CSV-based LF simulation for drone obstacle avoidance.

The simulation reads recorded ToF sensor data, feeds those readings into the avoid planner, and writes RC command outputs instead of sending commands to a flight controller.

## Main Files

```text
drone/simulation/src/drone.lf          # CSV replay entry point in this repository
drone/simulation/src/ToFBridgeCSV.lf   # CSV-backed ToF input reactor
```

## Data Locations

Repository-level data are stored in:

```text
data/drone/raw-logs/   # ToF datasets: Data1, Data2, Data3, Data4
data/drone/rc-out/     # RC command outputs generated by LF simulation
```

The current LF source may use paths such as `simulation/data/Data4/...` and `simulation/results/rc-out-4.csv`. Edit those LF parameters or copy data into the expected local simulation paths before running.

## Build and Run

From the `drone/` directory:

```bash
cd drone
lfc simulation/src/drone.lf
```

Run the generated executable according to the output location created by `lfc`.

## Switching Datasets

To replay another dataset, update the five `path=` parameters in `simulation/src/drone.lf`:

```lf
path="simulation/data/Data1/front.csv"
path="simulation/data/Data1/left.csv"
path="simulation/data/Data1/right.csv"
path="simulation/data/Data1/top.csv"
path="simulation/data/Data1/bottom.csv"
```

Also update the `log_path` parameter in the `MSPSender` instance so the RC output file is not overwritten.

## Plot the Generated RC Output

From the repository root:

```bash
python3 tools/plot_drone_path.py data/drone/rc-out/rc-out.csv data/drone/raw-logs/Data1
```

Compare original and obstacle-injected runs:

```bash
python3 tools/plot_drone_path_compare.py data/drone/rc-out/rc-out.csv data/drone/raw-logs/Data1 data/drone/rc-out/rc-out-2.csv data/drone/raw-logs/Data2 results/drone/simulation/original_vs_obstacle_injected_overlay.pdf
```

## Notes

- `MSPSender(port="")` disables serial output and logs RC commands to CSV.
- The simulation is useful for testing how ToF data changes propagate through planning and actuation.
- Use the generated trajectories as relative comparisons, not absolute position ground truth.
